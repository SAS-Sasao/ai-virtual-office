## モックの位置づけ（AI への指示）

- **UI/UX（見た目・情報設計・インタラクション）の参照仕様として扱う**。画面レイアウト・配色・状態表現・パネル構成は、要件定義 §8 より具体的な正とする
- **実装方式の参照にはしない**。モックはオフィス部分も DOM で描いているが、実装ではオフィス描画（部屋・家具・キャラ・吹き出し）は `apps/web/game/` の Canvas 2D、外枠（ヘッダ・サイドパネル・設定画面）が React（ui-dev 担当）である。モックの DOM 構造をそのまま React に移植しないこと
- **⚠ 一枚絵背景への全面移行は実装では採らない（ADR-002）**。本版のモックは `showCss:false` で部屋・家具の手続き描画を丸ごと無効化しているが、これをそのまま実装すると **FR-3（マスタからの間取り生成）と FR-6（エディタでの家具配置）が成立しない**。実装では「**手続き描画が正 / 背景画像は任意の装飾レイヤー**」のハイブリッド構成を採る。モックの `rooms` / `furniture` データ構造（座標・サイズ・トリガー語）は生きており、そちらがレイアウトの参照仕様である
- **モック内のダミーデータ（フロア名 Nebula Works / Comet Labs、キャラ名、部署構成）は仮**。実データは cc-sier-adapter が masters から生成する（FR-3）。ハードコードしない
- ピクセル素材はモック用プレースホルダー。実装時は CC0 / CC-BY 素材 + 出典表記（NFR-6）

## 抽出仕様 2: キャラクター状態の表現（FR-1 / 設計 §7 と対応）

| state | ラベル | 色 | 表現 |
|---|---|---|---|
| idle | 待機 | `#9aa0b8` | 吹き出しなし |
| type | 編集 | `#7ef29a` | 吹き出し |
| read | 読取 | `#6be5ff` | 吹き出し |
| terminal | 端末 | `#c39bff` | 吹き出し |
| browsing | 閲覧 | `#5aa2ff` | 吹き出し |
| thinking | 思考 | `#9aa0b8` | 吹き出し |
| waiting | 許可待ち | `#ffd166` | 吹き出しが blink 点滅 + 待ちパネル + ヘッダバッジ |
| done | 完了 | `#ffffff` | 吹き出し |

- モック未表現の `walk` / `leave` は移動アニメーション（Canvas 実装側の状態）として補完する
- **キャラ = PNG スプライト**（v2 の box-shadow ドット絵 + アウトラインから移行）。表示枠 52×56、`background-size:contain` / `background-position:bottom center` / `image-rendering:pixelated` / `filter:drop-shadow(1px 2px 0 rgba(0,0,0,.5))`。常時 bob（上下 2px・2.6s）。名前プレートは `rgba(11,13,24,.82)` 地
- **スプライトセットをフロアで使い分ける**: 1F Nebula Works = `office-*`（オフィス服）、2F Comet Labs = `rpg-*`。ロール単位ではなく**キャラ単位の固定割当**（モックの `SPR` マップ）。実装では masters のロールから決定論的に割り当てる
- 吹き出しは状態色の枠 + ハードシャドウ `1px 1px 0 rgba(0,0,0,.6)`。`waiting` のみ `blink 1s steps(2)` で点滅
- ホバーで詳細カード（z-index 20）: 名前(ロール) / 部署 / セッション · model / 状態 / ツール / 経過。カード枠は状態色
- サブエージェント: 会議室に spawn、親キャラと破線リンク（`#9aa0b8` dashed 4 4、z-index 4）、SubagentStop で退出
- セッション一覧クリック → 該当キャラへフォーカス（黄リング `3px solid #ffd166` / offset 2px / 3 秒）

## モックと実装のドリフト（要追従）

モックは仕様の探索が目的のため、実装の確定仕様と一致しない箇所がある。**実装側を正とする**もの:

| 箇所 | モックの表示 | 実装の確定仕様 |
|---|---|---|
| 設定画面の doctor 診断 | `hooks 導入 9/9（マーカー #ai-office）` | **8 イベント**、CLI 導入分のマーカーは **`#ai-office:cli`**（M1-2b）。本リポジトリ手書き分の `#ai-office` とは区別する |
| doctor のイベント到達判定 | `[OK]` 固定 | **警告であり exit code に影響しない**（未受信は正常な初期状態） |
