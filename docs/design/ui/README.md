## モックの位置づけ（AI への指示）

- **UI/UX（見た目・情報設計・インタラクション）の参照仕様として扱う**。画面レイアウト・配色・状態表現・パネル構成は、要件定義 §8 より具体的な正とする
- **実装方式の参照にはしない**。モックはオフィス部分も DOM で描いているが、実装ではオフィス描画（部屋・家具・キャラ・吹き出し）は `apps/web/game/` の Canvas 2D、外枠（ヘッダ・サイドパネル・設定画面）が React（ui-dev 担当）である。モックの DOM 構造をそのまま React に移植しないこと
- **モック内のダミーデータ（フロア名 Nebula Works / Comet Labs、キャラ名、部署構成）は仮**。実データは cc-sier-adapter が masters から生成する（FR-3）。ハードコードしない
- ピクセル素材はモック用プレースホルダー。実装時は CC0 / CC-BY 素材 + 出典表記（NFR-6）

## 抽出仕様 2: キャラクター状態の表現（FR-1 / 設計 §7 と対応）

| state | ラベル | 色 | 表現 |
|---|---|---|---|
| idle | 待機 | `#9aa0b8` | 吹き出しなし |
| type | 編集 | `#7ef29a` | 吹き出し |
| read | 読取 | `#6be5ff` | 吹き出し |
| terminal | 端末 | `#c39bff` | 吹き出し |
| browsing | 閲覧 | `#5aa2ff` | 吹き出し |
| thinking | 思考 | `#9aa0b8` | 吹き出し |
| waiting | 許可待ち | `#ffd166` | 吹き出しが blink 点滅 + 待ちパネル + ヘッダバッジ |
| done | 完了 | `#ffffff` | 吹き出し |

- モック未表現の `walk` / `leave` は移動アニメーション（Canvas 実装側の状態）として補完する
- キャラ = 10×14 ピクセルのドット絵（モックでは box-shadow 生成。実装ではスプライト）。常時 bob（上下 2px・2.4s）。名前プレート付き
- ホバーで詳細カード: 名前(ロール) / 部署 / セッション · model / 状態 / ツール / 経過。カード枠は状態色
- サブエージェント: 会議室に spawn、親キャラと破線リンク（`#9aa0b8` dashed）、SubagentStop で退出
- セッション一覧クリック → 該当キャラへフォーカス（黄リング 3 秒）

## 抽出仕様 3: 画面構成（要件 §8 と対応）

| 画面 | モックでの構成 | 要件 |
|---|---|---|
| 共通ヘッダ | ブランド / フロア切替タブ / ビュー切替（オフィス・リプレイ・エディタ・設定）/ 許可待ちバッジ（blink）/ 時計 / 音 ON・OFF | §8 |

| エディタ | 左パレット（デスク・床・壁・植物・本棚）+ 32px グリッドオーバーレイ + 「マスタ生成領域は再インポートで初期化」警告 + custom フラグ保存の説明 | エディタ P2・FR-6 |
| 設定 | 組織リポジトリ一覧（type バッジ: cc-sier / 汎用JSON）/ 通知（チャイム・タブバッジ）/ **doctor 診断 4 項目** / setup・doctor・teardown コマンド表示 | 設定 P2・§5.1 |

| ホワイトボード | 各フロアに 1 枚（150×44、白地 `#eef0f8`・見出し `#d94f5c`）。**実行中タスク**（idle 以外かつツール実行中のキャラ）を「名前: ツール」で最大 3 件表示、0 件なら「（なし）」。**ホバーでオフィス中央に 460px の拡大パネル**を表示し、全件を状態バッジ + 名前(ロール) + ツールで一覧（overlay z-index はキャラ・ホバーカードより上位） | オフィスビュー P1（実行中タスクの一覧性向上） |
