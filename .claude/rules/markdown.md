# Markdown ルール（.md）

対象によって編集規律が異なる:

1. **`.claude/skills/*/SKILL.md`**: 全面リライト禁止・**差分編集のみ**（設計書 §3.3）。「ギャップ記録」セクションを本文より先に維持し、実失敗が 3 件集まるまで本文を太らせない（§3.2）。リライトが不可避な場合は grep で必須セクションを事前抽出し新旧照合する
2. **`.claude/agents/*.md`**: 設計書 §4 / §4.1 / §4.2 と同期を保つ。office-qa の verdict JSON 契約（6 軸・致命軸・composite ≥ 0.85）を独断で変えない。**office-qa に Write / Edit を追加しない**（checker の構造的独立性）
3. **`docs/design/*.md`（設計正本）**: 変更時は冒頭のバージョン表を更新し、差分編集で行う。設計変更は「設計書を先に直す → 実装物（agents / skills / hooks）を同期」の順。実装だけ直して設計書を放置しない
4. **`docs/design/ui/`**: 正本は claude.ai/design プロジェクト。ミラー（dc.html）を手編集せず、デザイン側を更新して再インポートする（手順は同ディレクトリ README）
5. **`CLAUDE.md` / `README.md`**: 実装と乖離した記述を残さない。`.claude` 構成やコマンドを変えたら該当記述を同じ PR で更新する
